# LLM failure log

Журнал нужен, чтобы фиксировать забавные или опасные ошибки пользовательских объяснений и превращать их в guardrails для промпта, валидации и тестов.

## 2026-05-27

- В предсказании от GigaChat модель указала, что самолет, вероятно, не полетит, потому что "нет мест": "На рейс на Курильские острова 27 мая 2026 года вероятно нет мест". Это неверная интерпретация: сервис оценивает вероятность выполнения рейса, а не наличие билетов, мест, пассажиров или бронирований.
- Реакция: в prompt добавлен явный запрет на темы билетов, мест, пассажиров, бронирования, салона и продаж. В backend добавлена валидация LLM-ответа; при нарушении пользователь получает безопасное шаблонное объяснение.
- В предсказании на 18 августа 2026 года итог был `Да` при вероятности 47%, но объяснение начиналось с противоположного смысла: "Риск отмены или задержки рейса выше обычного". Это сбивает пользователя, потому что текст спорит с решением.
- Реакция: prompt усилен правилом согласованности с `decision`. В backend добавлена проверка противоречий: для `decision=yes` отклоняются объяснения, которые начинаются с повышенного риска отмены/невыполнения; для `decision=no` отклоняются объяснения с формулировками "скорее да", "высокий шанс" и т.п. Версия prompt/cache повышена, чтобы старые тексты не доставались из кэша.
